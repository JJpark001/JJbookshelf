# 个人书架 Web 应用（V4）

一个纯前端的个人书架管理工具，用于记录自己的待读 / 在读 / 已读书籍，支持封面、标签、评分、阅读进度等信息，并带有简单的用户操作日志。

本项目基于原来的单文件 Demo，做了模块化与工程化重构，方便后续长期维护和扩展。

---

## 功能概览

- **书籍管理**
  - 新增 / 编辑 / 删除书籍
  - 复制书籍（自动在原书名后追加数字后缀，如 `书名 (2)`、`书名 (3)`）
  - 行内编辑：在列表中**直接点击书名或作者名即可编辑**

- **书籍字段（标准化 Book 模型）**
  - `id`: 唯一标识（自动生成）
  - `title`: 书名
  - `author`: 作者
  - `cover`: 封面（支持 URL 或本地上传 base64）
  - `status`: 阅读状态（`未读` / `在读` / `已读`）
  - `tags`: 标签数组
  - `rating`: 评分（0–5 星）
  - `currentPage`: 当前页（≥ 0）
  - `totalPages`: 总页数（≥ 0）
  - `progress`: 阅读百分比（0–100，始终由当前页 / 总页数自动计算）
  - `createdAt` / `updatedAt`: 时间戳，毫秒

- **列表与筛选**
  - 搜索：按书名 / 作者 / 标签模糊搜索
  - 状态筛选：全部 / 未读 / 在读 / 已读
  - 排序：
    - 最新添加 / 最早添加
    - 评分从高到低
    - 进度从高到低
    - 书名 A-Z
  - 分页：每页 10 本，可上一页 / 下一页

- **阅读进度**
  - 使用 `当前页 / 总页数` 计算百分比并展示进度条
  - 当编辑页数或总页数时，进度百分比自动重新计算

- **统计与视图**
  - 顶部统计卡：
    - 书籍总数
    - 未读 / 在读 / 已读数量
    - 已读比例
    - 平均评分（星星展示）
  - 右侧侧边栏：
    - 阅读状态汇总
    - 小提示卡片
    - **最近操作日志**（仅保留最近若干条）

- **用户体验**
  - 暗色 / 浅色主题切换（记忆在 localStorage）
  - 统一的弹窗表单，带基础校验（书名必填、页数范围检查等）
  - 列表行内操作按钮（编辑 / 复制 / 删除）
  - 搜索输入不会打断光标（只局部刷新表格）

- **本地存储与数据迁移**
  - 使用 `localStorage` 持久化：
    - 书籍数据：`my-bookshelf-books`
    - 元信息：`my-bookshelf-meta`
    - 操作日志：`my-bookshelf-logs`
  - 使用版本化数据结构，支持从早期“仅数组”的数据格式迁移

## 使用方式

### 1. 直接双击运行（推荐给普通使用者）

1. 打开资源管理器，进入项目目录：`d:\AIAIAIAIAI\在线书架`
2. 双击 `index.html`，用浏览器（Chrome/Edge 等）打开
3. 就可以开始：
   - 点击右上角“添加”录入书籍
   - 在列表中搜索、筛选、排序、分页
   - 点击书名 / 作者进行行内编辑
   - 右侧查看最近的操作日志

> 所有数据都保存在本机浏览器的 `localStorage` 中，不会上传到任何服务器。

### 2. 使用本地静态服务器（给开发者）

如果你安装了 Node.js，可以在项目目录中启动一个本地静态服务器，方便未来扩展：

```bash
cd "d:\AIAIAIAIAI\在线书架"
npx serve .
```

然后在浏览器访问：

```text
http://localhost:3000
```

## 关键技术设计

- **模块化**
  - `bookModel.js`：只负责数据结构、创建 / 更新逻辑、统计和进度计算，不依赖 DOM
  - `storage.js`：只负责与 `localStorage` 的读写与版本迁移
  - `main.js`：组织应用状态、渲染 DOM、处理交互事件

- **数据版本与迁移**
  - 数据持久化格式带有 `version` 字段
  - 加载时：
    - 先尝试解析 `{ version, items }`
    - 若失败，再回退到旧的纯数组格式
  - 每本书在加载时都会通过 `migrateBook` 标准化，避免旧数据字段缺失或类型不一致影响功能

- **进度与评分**
  - `progress` 字段不直接从表单信任，而是统一通过：
    - `computeProgress(currentPage, totalPages)` 计算
  - 编辑书籍时会丢弃旧的 `progress`，强制根据最新页数重新计算，确保展示正确

- **操作日志**
  - 通过 `pushLog(message)` 记录关键操作：
    - 添加 / 编辑 / 删除 / 复制书籍
    - 行内编辑书名 / 作者
  - 日志结构：
    - `id`: 日志 ID
    - `message`: 文本描述
    - `time`: 时间戳
  - 保留最近 50 条，UI 中显示最近 10 条

## 浏览器兼容性

- 主要针对 **现代桌面浏览器**（Chrome / Edge / Firefox 最新版）
- 使用了一些 ES6+ 特性（如 `const/let`、模板字符串、解构等），在旧版 IE 上不会工作

如果你在某个浏览器上遇到问题（例如页面空白或功能异常），可以：

- 按 F12 打开「开发者工具」查看控制台错误
- 或将错误截图、描述发给开发者协助排查
